## Design Context

### Users
VaultFleet is used by operators and server owners who manage backup agents,
storage targets, policies, task history, snapshots, notifications, and system
maintenance from a web console. Mobile usage is most likely quick inspection,
urgent troubleshooting, checking backup status, copying install commands, and
triggering focused actions while away from a desktop.

### Brand Personality
Quiet, reliable, and operational. The interface should feel like a trustworthy
control plane: dense enough for scanning, restrained in decoration, and clear
about status, risk, and irreversible actions.

### Aesthetic Direction
Use a utilitarian Ant Design console aesthetic with a light background, compact
cards, clear hierarchy, and strong status affordances. Mobile should keep the
same information architecture while adapting navigation, tables, forms, and
drawers for touch and narrow viewports.

### Design Principles
- Preserve core functionality on mobile; adapt layout instead of hiding tasks.
- Prefer single-column mobile flows with full-width controls and 44px touch
  targets for primary actions.
- Keep data-heavy tables usable with intentional horizontal scrolling and
  less-important columns hidden at small breakpoints.
- Make operational status easy to scan through badges, concise headings, and
  predictable page actions.
- Prevent accidental destructive actions with clear confirmations that remain
  readable on small screens.
