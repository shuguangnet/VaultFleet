# 节点标签与批量策略下发

VaultFleet 支持用节点标签组织多节点环境，并把一条已有备份策略克隆下发到多个目标节点。该能力适合 OpenStack、VPS 集群、多地域节点或按业务分组的服务器批量运维。

## 节点标签

在 **节点管理** 页面可以为节点维护标签，例如：

- `prod`
- `stage`
- `web`
- `db`
- `openstack:az1`
- `region:sg`

标签会被规范化为小写、去重后的列表。每个节点最多 20 个标签，每个标签最长 40 个字符，支持字母、数字、`.`、`_`、`-` 和 `:`。

节点列表可以按多个标签筛选。筛选使用交集匹配：选择 `prod` 和 `web` 时，只返回同时包含两个标签的节点。

## 批量策略下发

在 **备份策略** 页面，打开某条策略的操作菜单并选择 **批量下发**。批量下发会：

1. 读取当前策略作为源策略。
2. 解析显式选择的目标节点和标签匹配到的节点。
3. 为每个目标节点创建一条普通备份策略。
4. 将新策略标记为未同步，复用现有 Agent 策略推送链路。
5. 返回每个目标节点的成功或失败结果。

每个目标节点的新策略都会使用独立仓库路径：

```text
vaultfleet/<target-agent-id>
```

这可以避免多个节点误写到同一个 restic 仓库路径。批量下发不会创建动态策略组；后续新加入的同标签节点不会自动继承已有批量下发结果，需要再次执行批量下发。

## 注意事项

- 批量下发克隆的是已有策略配置，包括存储、备份源、排除规则、钩子、调度、保留策略、传输参数、任务超时和可恢复性验证设置。
- 如果源策略包含 Docker 容器备份源，目标节点必须支持 Docker 工作负载备份能力；不支持的节点会在结果中标记失败，其它节点仍可成功创建策略。
- VaultFleet 不会自动判断目标节点上路径是否存在。首次批量下发后，建议运行一次备份或可恢复性验证来确认配置。
- 只读用户不能更新节点标签或执行批量下发。管理员和运维角色按现有 RBAC 权限控制操作。
