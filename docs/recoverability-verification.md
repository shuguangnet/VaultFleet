# 备份可恢复性验证

VaultFleet 支持对 snapshot 模式的 restic 仓库执行可恢复性验证，用于降低“备份任务成功但恢复时才发现仓库不可用”的风险。

## 验证内容

验证任务会在 Agent 本机使用当前策略的 rclone/restic 配置访问仓库，并针对最新快照执行：

- `restic snapshots`：确认仓库可访问并能找到快照。
- `restic check`：检查 restic 仓库元数据和索引。
- `restic ls <snapshot>`：确认最新快照内容可枚举。
- 抽样 `ls`：对快照中的样本路径做二次读取。
- 可选小文件恢复：把一个样本文件恢复到 Agent 临时目录并清理。

验证结果会以 `verify` 类型任务写入任务历史，并在策略列表显示最近验证状态。

## 使用建议

- 对重要策略启用每日或每周验证，例如 `0 4 * * *`。
- 大仓库的 `restic check` 可能耗时较长，应设置合适的超时时间。
- 小文件恢复测试会写入 Agent 临时目录，默认关闭；在磁盘空间紧张的节点上谨慎开启。
- 存储迁移、凭据轮换、Agent 升级后，建议手动执行一次“立即验证”。

## 限制

可恢复性验证不是应用一致性保证。它不能证明数据库事务一致、容器能成功启动、外部依赖可用或业务数据语义正确。

可恢复性验证也不替代恢复预检。验证证明备份仓库和快照在验证时间点可读；恢复预检仍用于执行恢复前检查目标节点、目标路径、Docker 冲突和 Agent 能力。

当前版本仅支持 snapshot/restic 策略。压缩包归档模式需要不同的下载和解包校验流程，暂不支持。
