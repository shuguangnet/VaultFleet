# 备份调度与保留规则

## 执行计划

策略编辑器提供以下常用模式，保存时仍使用兼容现有 API 和 Agent 的 `schedule` 字符串：

- 每天：选择固定时间，例如每天 02:30 生成 `30 2 * * *`。
- 每周：明确选择一个或多个星期和执行时间，例如周一、周三、周五 03:00 生成 `0 3 * * 1,3,5`。
- 每月：选择每月日期和执行时间，例如每月 15 日 01:00 生成 `0 1 15 * *`。
- 固定间隔：按分钟或小时执行，例如每 6 小时生成 `0 */6 * * *`。
- 自定义 Cron：保留高级表达式，不会把无法无损识别的已有表达式自动改写。

Master 和 Agent 接受五段 Cron、包含秒字段的六段 Cron，以及 robfig/cron 支持的描述符，例如 `@daily` 和 `@every 6h`。可视化模式只生成标准五段 Cron。

所有计划均以目标 Agent 的本地时间执行。不同节点处于不同时区时，同一个 Cron 可能对应不同的绝对执行时间；当前版本不提供策略级独立时区。

## 保留规则

保留配置继续使用四个兼容字段：

- `keep_last`：保留最近 N 次备份。
- `keep_daily`：为最近 N 天各保留一个代表性快照。
- `keep_weekly`：为最近 N 周各保留一个代表性快照。
- `keep_monthly`：为最近 N 月各保留一个代表性快照。

四项规则取并集，不是简单相加。同一个快照同时满足多项规则时只占一份。值 `0` 表示关闭该层级，至少一项必须大于 `0`。

例如 `{ "keep_last": 7, "keep_daily": 7, "keep_weekly": 4, "keep_monthly": 0 }` 会保留最近 7 次备份，并补充每日和每周的代表性时间点；它不保证恰好保留 18 份。
