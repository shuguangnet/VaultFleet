## Why

备份策略目前直接暴露 Cron 表达式和 restic 保留字段，用户难以确认“什么时候执行”以及“究竟保留多少份”。需要把常见调度与保留需求转换为清晰的业务语言，同时保留高级用户直接编辑 Cron 的能力。

## What Changes

- 增加可视化调度构建器，支持每天、每周指定多个星期、每月指定日期、按小时或分钟间隔执行，并选择具体执行时间。
- 保留“自定义 Cron”高级模式，实时校验五段式表达式并显示人类可读说明和后续执行时间预览。
- 编辑已有策略时自动识别可视化支持的 Cron；无法识别的表达式自动进入自定义模式，且不改写原值。
- 重构保留策略文案与布局，明确区分“保留最近 N 次”“每日/每周/每月保留 N 个代表性快照”。
- 保留现有保留预设，同时允许用户随时进入自定义模式填写每个数量；`0` 明确表示不启用该层级。
- 在保存前校验调度与保留规则，并提供可读摘要，避免无效 Cron 或所有保留数量均为零。
- 保持现有 `schedule` 字符串及 `retention` 字段兼容，不要求数据库迁移。

## Capabilities

### New Capabilities

- `policy-scheduling-retention`: 备份策略的可视化调度、自定义 Cron、执行时间预览及清晰的快照保留规则配置。

### Modified Capabilities

- None.

## Impact

- Web UI：策略创建/编辑抽屉、Cron 工具函数、策略类型与前端测试。
- Master API：增加 Cron 和保留配置的服务端校验，继续接受现有请求结构。
- Agent 与调度器：不改变协议字段；继续消费最终生成的标准五段式 Cron 和现有保留规则。
- 文档：补充调度模式、Cron 时区语义和多层保留规则示例。
